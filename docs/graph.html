関連の読み方

  • implements — 各 adapter は Source を、各 writer は Sink を実装する。
  • produces / 入力 — Source → RawRecord → ②Transform → SinkBatch → ③Sink のデータフロー。
  • kind=rdb/kg/ft — SinkBatch は種別で対応 sink へ振り分けられる（runner のルーティング）。
  • 破線 — opt-in / core 外境界（LLM 利用・FT runner 起動・raw verbatim・②の所在）。

色の濃淡: 塗りが薄いノードは型/インターフェース（契約）、濃いノードは具象クラス/関数、破線円は core 外の存在。 関数・型の詳細なシグネチャとパラメータは API リファレンス を参照。
