core — 契約 core

▶interface Source<C> · name; crawl(config: C): Promise<RawRecord[]>

①Crawl の抽象契約。取得元から RawRecord[] を得る。adapter はこれを実装する。

メンバ型説明
namestringadapter 名。manifest の crawl.sources[].adapter と突合。
crawl(config)(C) => Promise<RawRecord[]>config は adapter 固有。
▶interface Transform<O> · name; run(records: RawRecord[]): Promise<O[]>

②Clean/Categorize の契約。RawRecord[] を sink ネイティブのエンベロープへ整形する。 1 入力から複数 sink 向けエンベロープを出してよい。実体はサービスリポ側（共有 lib は契約のみ）。

▶interface Sink<P> · name; accepts: SinkKind; write(payload: P): Promise<void>

③Save の契約。渡されたものを保存先へ verbatim 書込。accepts が 'raw' の sink は RawRecord[] を、それ以外は対応する SinkBatch を受ける。

メンバ型説明
accepts'raw'|'rdb'|'kg'|'ft'受け付けるペイロード種別。runner のルーティングに使う。
▶type RawRecord

①→②/①→raw の受け渡し型。source 非依存の正規化生レコード。

フィールド型説明
sourcestring取得元種別（例 "notion"）。
sourceIdstringsource 内一意 ID（dedup キー）。
fetchedAtstring取得時刻（ISO8601 / UTC）。
url? / title? / text?stringtext = 正規化本文（②の主入力）。
rawunknownsource 原データ verbatim（replay / 監査）。
meta?Record<string,unknown>source 固有の補助情報。
▶type SinkBatch = RdbBatch | KgBatch | FtBatch

②が出す sink ネイティブ「最終形」エンベロープ。kind で ③へ振り分ける。

種別形
RdbBatch{ kind:'rdb'; upserts: { table; rows; conflictKey }[] }
KgBatch{ kind:'kg'; nodes: GraphNode[]; edges: GraphEdge[] }
FtBatch{ kind:'ft'; task; examples: FtExample[]; dataset?; model? }

関連型: Row, RdbUpsert, GraphNode, GraphEdge, NodeRef, FtTask, FtClassificationExample, FtChatExample, FtPromptExample, FtExample, SinkKind。

▶type Manifest · SourceSpec · SinkSpec

パイプライン宣言。サービスはコードでなくこの形で流れを定義する。

フィールド型説明
pipelinestringパイプライン名（ログ/レポート用）。
crawl.sourcesSourceSpec[]{ adapter; config } の配列。
crawl.rawSave?SinkSpec①出力を verbatim 保存する raw sink（任意）。
transform?string②の adapter 名。省略時は raw 取得のみ。
save?SinkSpec[]③保存先（複数可）。{ adapter; accepts; config? }。
replayFrom?SinkSpec再処理モード。raw store から読み直す。
▶function loadManifest(filePath: string): Promise<Manifest>

YAML（.yaml/.yml）または JSON ファイルを Manifest として読み込む。 pipeline（string）と crawl（object）の存在を検証し、無ければ throw。

runner — 実行 runner

▶function runPipeline(manifest: Manifest, deps: PipelineDeps): Promise<RunReport>

manifest を解釈して 3 ステージを実行する。crawl(①) → [rawSave] → transform(②) → save(③)。 実体は deps として注入する（adapter 名 → インスタンス）。replayFrom 指定 + replayLoader があれば replay モード。

PipelineDeps

キー型説明
sourcesRecord<string, Source>必須。adapter 名 → Source。
transforms?Record<string, Transform>②の名前解決用。
sinksRecord<string, Sink>必須。adapter 名 → Sink。
replayLoader?(spec) => Promise<RawRecord[]>replayFrom の起点読込。

RunReport（戻り値）

{ pipeline; mode:'crawl'|'replay'; recordCount; rawSaved; written: {sink;kind;batches}[]; errors: StageError[] }。 sink の accepts と manifest の宣言が食い違うと throw。各ステージのエラーは errors に集約（致命でないものは継続）。

① Crawl adapter crawl

▶class NotionSource implements Source · name="notion"

Notion DB を再帰クロール（公式 REST API・token 必要）。Tirocinium packages/notion から移植。

config: NotionSourceConfig

パラメータ型既定 / 説明
databaseIdstring必須。対象 Notion database id。
token?stringintegration token。省略時 env NOTION_TOKEN。
crawl?.maxDepthnumber既定 3。row=0 から潜る深さ。
crawl?.maxPagesnumber既定 500。安全弁。
crawl?.includeChildDatabasesboolean既定 true。child_database の row も辿る。

deps（コンストラクタ）: { api?: NotionApi; now?: () => string }（テスト/再利用で差し替え）。

関連エクスポート: crawlDatabase(api, databaseId, opts), NotionApiClient, createNotionClient(config), blocksToMarkdown, extractTitle, simplifyProperties。

▶class NotionPublicSource implements Source · name="notion-public"

公開 Notion ページを Playwright でレンダリング取得（API token 不要）。1 URL → 1 RawRecord。

config: NotionPublicSourceConfig

パラメータ型既定 / 説明
urlstring必須。公開 Notion ページ URL。
options?.timeoutnumber既定 30000 ms。
options?.scrollDelaynumber既定 800 ms。スクロール間待機。
options?.maxScrollsnumber既定 15。lazy-load 展開回数。

deps: { fetcher?: PageFetcher; now? }。関連: PlaywrightFetcher（playwright を動的 import）。

▶class YouTubeSource implements Source · name="youtube"

YouTube Data API v3 の commentThreads をページネーション取得。

config: YouTubeSourceConfig

パラメータ型既定 / 説明
videoIdstring必須。
apiKeystring必須。API キー。
maxResults?number既定 100。最大取得コメント数。
textFormat?'plainText'|'html'既定 'plainText'。

deps: { fetch?; now? }。

▶class RedditSource implements Source · name="reddit"

Reddit 公開 JSON API で投稿一覧 or 投稿コメントを取得（認証不要・User-Agent 必須）。

config: RedditSourceConfig

パラメータ型既定 / 説明
mode?'posts'|'comments'既定 'posts'。
subreddit?stringmode=posts で必須（r/ なし）。
postId?stringmode=comments で必須。
subredditForPost?stringcomments 取得時の subreddit。
sort?'hot'|'new'|'top'|'rising'既定 'hot'（posts のみ）。
limit?numberposts 既定 100 / comments 既定 500。
▶class WebsiteSource implements Source · name="website"

URL から HTML を取得。HTML→構造化テキストは HtmlParser を注入（Lector 等）。未注入時はタグ除去 + タイトル/同一ドメインリンク抽出のフォールバック。

config: WebsiteSourceConfig

パラメータ型既定 / 説明
urlstring必須。
userAgent?string既定 Canalis-Crawler/1.0。
timeoutMs?number既定 10000。

deps: { fetch?; now?; htmlParser?: HtmlParser }。

Discord 1 チャンネルのメッセージを取得（REST API v10・raw fetch・429 リトライ）。

▶class DiscordGuildSource implements Source · name="discord-guild"

ギルド全テキスト/アナウンスチャンネル + フォーラムスレッド（アクティブ + アーカイブ）をクロール。1 チャンネル/投稿につき 1 RawRecord（会話はタイムライン化）。

config: DiscordGuildSourceConfig

③ Save writer save

▶class JsonlRawSink implements Sink · accepts="raw"

RawRecord[] を 1 行 1 レコードの JSONL で verbatim 保存。node の fs のみ依存。

config型説明
dirstring必須。出力ディレクトリ。
stamp?stringファイル名のタイムスタンプ（省略時は呼出時刻）。

replay 用: loadRawRecordsFromFile(file) / loadRawRecordsFromDir(dir)（*.jsonl を時系列連結）。

▶class PostgresSink(exec: SqlExecutor, name?) implements Sink · accepts="rdb"

RdbBatch を INSERT … ON CONFLICT DO UPDATE に落として upsert。識別子は安全に引用。 DB ドライバは注入（共有 lib は postgres に直接依存しない）。

引数型説明
execSqlExecutor = (query, params) => Promise<void>パラメタ化クエリ実行関数。
name?string既定 "postgres"。
▶class KuzuSink(exec: CypherExecutor, config?, name?) implements Sink · accepts="kg"

KgBatch の node/edge を Kuzu へ MERGE。driver は注入。対象スキーマは事前作成済が前提（サービスの責務）。

引数型説明
execCypherExecutor = (query, params) => Promise<void>Cypher 実行関数。
config?.keyPropstring既定 "key"。MERGE キーのプロパティ名。
name?string既定 "kuzu"。
▶class FtSink(config: FtSinkConfig, name?) implements Sink · accepts="ft"

FtBatch を <dir>/<dataset>/<stamp>/ に data.jsonl + job.json として materialize し、 任意で外部 FtRunner を起動。core は学習を持たない（LLM/ML 依存ゼロ）。

config型説明
dirstring必須。出力ルート。
runner?FtRunner = (job: FtJob) => Promise<void>未指定なら materialize のみ。
now?() => stringstamp の時刻源。

commandRunner(command, args?, {cwd}?) — 外部コマンドを <command> [...args] --job <job.json> で spawn する FtRunner ファクトリ（shell 非経由・exit 購読）。 FtJob: { task; dataset; count; dataPath; jobPath; model; createdAt }。

LLM（opt-in） llm @ludiars/canalis/llm

root には含まれない。 LLM を使うサービス② が明示的にこのサブパスから import して DI する。 失敗（未起動/HTTPエラー/空応答）は throw → ②側で握って決定論フォールバックに倒す（LLM は従、heuristic が主）。
▶function localOpenAiExecutor(opts): LlmExecutor

OpenAI 互換 /v1/chat/completions を叩く注入ヘルパ（Ollama / vLLM / LM Studio / llama.cpp）。SDK 依存も API キー要求も無い（raw fetch + baseUrl のみ）。

opts型既定 / 説明
baseUrlstring必須。末尾 /v1（例 http://localhost:11434/v1）。
modelstring必須。既定モデル（req.model が優先）。
apiKey?string設定時のみ Authorization: Bearer。
timeoutMs?number既定 120000。
fetchImpl?typeof fetchテスト差し替え。

LlmExecutor = (req: LlmRequest) => Promise<LlmResponse>。 LlmRequest = { prompt; system?; model?; maxTokens?(既定4096); timeoutMs? } / LlmResponse = { text; usage? }。

▶class Cascade(tiers: TierScorer[], options?) · score / scoreBatch

段を束ね、安い順に評価し確信が出た段で打ち切る分類カスケード。tiers = [Tier0 決定論, Tier1 ローカルモデル, Tier2 LLM]。各段が null（abstain）なら次段へ。

メンバ型説明
options.minConfidence?number既定 0。採用に必要な最小 confidence。
score(input)Promise<ScoreResult|null>1 件評価。解決段の index を tier に格納。
scoreBatch(inputs)Promise<(ScoreResult|null)[]>並行評価。

toClassificationExample(text, result) — カスケード結果を FT(分類) の { input, label } 学習例に変換（自己改善ループ）。

▶function lexiconScorer(config: LexiconConfig): TierScorer

Tier0: 辞書 + 信号の決定論スコアラ（LLM/外部依存なし・純関数）。用語/emoji のラベル別重みを合算し最大ラベルを返す。閾値未満は abstain。

config型既定 / 説明
termsRecord<string, WeightMap>必須。用語 → ラベル別重み。
emoji?Record<string, WeightMap>emoji → ラベル別重み。
minWeight?number既定 1。採用最小合計重み。
caseInsensitive?boolean既定 true。
▶function executorScorer(opts): TierScorer

Tier1/2: 注入 LlmExecutor を分類用 TierScorer に橋渡し。候補外応答は abstain（null）→ 次段。

opts型既定 / 説明
executorLlmExecutor必須。例 localOpenAiExecutor(...)。
labels?string[]候補ラベル既定（input.labels が優先）。
system? / model?stringsystem 指示 / モデル上書き。
confidence?number既定 0.85（LLM は確率を返さないため固定）。

関連型: TierScorer, ScoreInput {text; signals?; labels?}, ScoreResult {label; confidence; tier; scores?}, Signals。

config config

▶function resolveNotionToken / applyNotionConfigToEnv / getConfigStatus / configPath

リポ内 config.json による設定。notion.token は salt 付き AES-256-GCM 暗号化、非シークレット（version / minIntervalMs）は平文。master 鍵は env CANALIS_MASTER_KEY → 無ければマシン束縛値。

関数戻り値説明
resolveNotionToken()string | nullenv NOTION_TOKEN 優先 → 無ければ config の復号トークン。
applyNotionConfigToEnv()void非シークレット設定を env へ注入（既存は上書きしない）。
getConfigStatus()ConfigStatuspath / tokenSource / マスク表示 hint / version / minIntervalMs。
configPath()stringenv CANALIS_CONFIG_PATH → リポ直下 config.json。

CLI bin: canalis

▶$ canalis notion-crawl <databaseId> [--out <dir>] [--max-depth N] [--max-pages N]

②/③ のサービス固有コード無しで動く範囲 = ①Crawl + raw 保存 の standalone（Tirocinium scripts/notion-crawl 置換）。 token は env NOTION_TOKEN 優先 → 無ければ config の暗号化トークン。結果は <out>/notion/<databaseId>/<timestamp>.jsonl。

▶$ canalis config set|unset|show|path …
サブコマンド用途
config set notion.token [value]token 暗号化保存（value 省略で非表示入力 / パイプ可）。
config set notion.version <v>Notion-Version（平文）。
config set notion.minIntervalMs <ms>レート間隔 ms（平文）。
config unset notion.tokentoken 削除。
config show設定一覧（token マスク）。
config pathconfig.json のパス。
